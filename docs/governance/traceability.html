The current registry derives from jmMSE, not the SCW17 test table. It does not assert that MP29 or MP32 has been formally recommended or adopted. The six additional CMPs are comparison cases, not advanced candidates.

2 Information that must accompany each CMP

Every CMP version must link to:

  • the method used to calculate its indicator, harvest control rule, and catch allocation;
  • all parameter values and annual data requirements;
  • the exact saved version of jmMSE used for evaluation;
  • OM and robustness scenario definitions;
  • definitions of the performance measures and links to the saved results;
  • review findings and known limitations; and
  • the decision that changed its status.

3 Types of change

Class Examples Required review
Wording only Spelling, navigation, or clearer background explanation Documentation review
Same calculation, revised code Software change shown to give the same answers Code review plus a repeat-calculation test
Scientific Parameter, index, window, HCR, constraint, allocation, or fallback change Renewed scientific review and normally MSE evaluation
Status Test to shortlist, recommendation, adoption, or retirement Cited decision record

4 Work still needed

  • Confirm the formal working-group status and stable identifiers for MP29 and MP32.
  • Reconcile the 100-iteration MP29/MP32 tuned triggers with the 500-iteration MP41/MP42 sensitivity values before approving a final version.
  • Archive the full jmMSE run objects alongside the published Slick file and its digital fingerprint.
  • Complete the required units, annual cutoffs, missing-data rules, rounding, and exceptional-circumstances provisions.
  • Add saved worked examples and automatic repeat-calculation checks.
